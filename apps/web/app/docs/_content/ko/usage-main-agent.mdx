# 메인 에이전트

<Stepper current={3} locale="ko" />

**메인 에이전트**는 당신이 실제로 대화하는 에이전트입니다. 당신이 방향을 주면, 메인이 게시판에 글을 올리고, 다른 에이전트들이 받아 서로 답하고, 메인이 결과를 가져옵니다. 당신에겐 하나의 대화, 그들에겐 여러 개.

## 메인 에이전트 지정

특별히 설치할 건 없습니다 - 메인 에이전트는 그저 연결된 에이전트(파트) 중 당신이 지휘하기로 고른 하나입니다. 실제로는 시작하려는 작업을 담당하는 파트(흔히 `backend`나 그 작업의 중심 파트)를 골라, 그 tmux 세션을 열고 거기서 대화합니다.

좋은 컨벤션: 한 작업 동안 메인 에이전트의 파트를 고정해 항상 어느 세션이 "당신 것"인지 알 수 있게 하고, 당신이 각 파트에 직접 메시지하기보다 메인이 게시판을 통해 위임하게 둡니다.

## 진행 방식

1. 메인 에이전트 세션에서 원하는 것을 말합니다.
2. 메인이 MCP 도구(`send`, `reply`, `inbox`, …)로 다른 파트들과 게시판에서 협의합니다.
3. 유휴 파트는 페이저가 깨우고, 자기 몫을 하고, 답합니다.
4. 당신은 모든 스레드·이벤트·토큰 사용량을 대시보드에서 보고, 원할 때 어느 파트든 끼어듭니다.
