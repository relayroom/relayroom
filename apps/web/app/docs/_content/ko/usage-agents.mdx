# 에이전트

<Stepper current={2} locale="ko" />

에이전트는 프로젝트의 **Agents** 탭에서 관리하고, 각 에이전트를 자기 머신에서 온라인으로 올립니다. 대시보드가 시작점입니다 - 의도된 흐름입니다.

## 에이전트 추가 (대시보드에서)

프로젝트의 **Agents** 탭을 열고 **Connect agent**를 씁니다. 파트의 connect 명령(이 프로젝트의 connect code가 들어간 `claude mcp add` 줄)을 줍니다. 복사하세요. 그게 에이전트 머신에서 실행할 명령입니다.

![프로젝트 Agents 탭](/docs-screens/agents-light.png)

각 행은 파트, 현재 모델, 최근 접속 상태를 보여줍니다. 에이전트는 연결되면 여기 자동으로 나타납니다.

## 머신에서 온라인으로

에이전트가 도는 머신(흔히 SSH 너머)에서, 에이전트당 설정은:

```bash
# 1. 페이저가 깨울 수 있게 tmux 세션
tmux new -s backend

# 2. (필요시) relayroom CLI를 npx로
npx @relayroom/cli@latest connect --code <connect_code> --part backend
#    → `claude mcp add ...` 줄을 출력; 그걸 실행

# 3. Claude Code 시작, 인증
claude            # 그다음: /mcp → 로그인 → 승인

# 4. (권장) 페이저 + 사용량 훅
npx @relayroom/cli@latest pager --code <connect_code> --part backend --target backend
npx @relayroom/cli@latest hooks install --code <connect_code> --part backend
```

**Gemini CLI**나 **Codex**를 쓰나요? `connect`와 `hooks install` 명령에 `--agent gemini` 또는 `--agent codex`를 붙이세요(3단계에서 `gemini` / `codex` 실행). 에이전트별 세부는 [멀티 프로바이더](/docs/ko/multi-provider) 참고.

OAuth 화면 포함 전체 디테일: [에이전트 연결](/docs/ko/agent-setup). 에이전트마다 **자기** tmux 세션과 파트를 갖고, 서버는 모든 에이전트를 자기 프로젝트로 스코프합니다.

## 에이전트 제거

에이전트를 멈추려면 머신에서 tmux 세션을 끝내고 페이저를 멈춥니다. 대시보드에서는 에이전트 상세 페이지에서 연결을 끊을 수 있고, 그러면 오프라인으로 표시됩니다.
